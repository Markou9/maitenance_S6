# Pixel Tracer - Maintenance Applicative

## 1. Introduction

### 1.1 Contexte du projet

Ce TP s’inscrit dans le cadre du module de Maintenance Applicative. L’objectif est de nous initier aux techniques de rétro-conception, de redocumentation et de refactoring d’une application existante.

Le projet étudié est **Pixel Tracer**, un logiciel de dessin vectoriel écrit en langage C. Il permet de créer et de manipuler des formes géométriques telles que des points, des lignes, des cercles, des polygones et des courbes de Bézier dans un environnement de dessin en mode texte, basé sur un affichage ASCII.

### 1.2 Démarche générale

Ce travail constitue la première phase d’un projet de maintenance organisé en trois étapes :

- **Rétroconception (TP1)** : analyse du code existant et ajout de documentation.
- **Récupération du modèle de conception (TP2)** : identification des concepts métier et transformation vers un modèle orienté objet.
- **Refactoring (TP3)** : réécriture en Java avec amélioration de la structure du code.

### 1.3 Ressources

Les ressources utilisées pour ce travail sont :

- le code source original du projet Pixel Tracer en langage C ;
- les fichiers d’en-tête permettant d’identifier les structures et fonctions principales ;
- le dépôt Git du projet ;
- les outils de développement utilisés, notamment VS Code, Git, GitHub et Maven.
